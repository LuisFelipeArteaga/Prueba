Q2. ¿Qué es el sesgo de selección? (What is Selection Bias?)¶

Se utiliza cuando se realiza un análisis en un subconjunto de datos con el fin de sacar una conclusión sobre la población, pero las conclusiones obtenidas probablemente serán incorrectas, porque el subconjunto difiere de la población de alguna manera importante. El sesgo de selección se introduce como un error en el muestreo y tener una selección para el análisis que no está debidamente aleatorizada.

Q3. ¿Qué es la compensación de sesgo-varianza? (What is bias-variance trade-off?)¶

Si tenemos que un modelo es demasiado simple y tiene muy pocos parámetros, este puede tener un alto sesgo y una baja varianza. Si, por el contrario, si nuestro modelo tiene una gran cantidad de parámetros, entonces tendrá una alta varianza y su sesgo bajo. Por lo cual es pertinente encontrar un equilibrio en que nuestro modelo en donde no se sobreajuste ni desajusten los datos. Esta compensación en la complejidad del modelo es la razón por la que existe una compensación entre el sesgo y la varianza.

Q25. Explique cómo funciona una curva ROC. (Explain how a ROC curve works?)¶

ROC) es un diagrama gráfico que se utiliza para mostrar la capacidad de diagnóstico de los clasificadores binarios. Una curva ROC se construye trazando la tasa de verdaderos positivos (TPR) contra la tasa de falsos positivos (FPR). La tasa de verdaderos positivos es la proporción de observaciones que se predijo correctamente que serían positivas de todas las observaciones positivas (TP / (TP + FN)). De manera similar, la tasa de falsos positivos es la proporción de observaciones que se predice incorrectamente como positivas de todas las observaciones negativas (FP / (TN + FP)).

La curva ROC muestra el compromiso entre sensibilidad (o TPR) y especificidad (1 - FPR). Los clasificadores que dan curvas más cercanas a la esquina superior izquierda indican un mejor rendimiento. Como línea de base, se espera que un clasificador aleatorio dé puntos a lo largo de la diagonal (FPR = TPR). Cuanto más se acerque la curva a la diagonal de 45 grados del espacio ROC, menos precisa será la prueba.

Para comparar diferentes clasificadores, puede ser útil resumir el desempeño de cada clasificador en una sola medida. Un método común es calcular el área bajo la curva ROC, que se abrevia como AUC.

Un mejor ajuste del conjunto de datos de entrenamiento en comparación con el conjunto de datos de prueba generalmente apunta a un sobreajuste. Por tanto, un conjunto de prueba es un conjunto de ejemplos que se utilizan únicamente para evaluar el rendimiento es decir, la generalización de un clasificador completamente especificado. Para hacer esto, el modelo final se usa para predecir clasificaciones de ejemplos en el conjunto de prueba. Esas predicciones se comparan con las clasificaciones reales de los ejemplos para evaluar la precisión del modelo.

Q39. Explique la validación cruzada. (Explain cross-validation)¶

La validación cruzada es una técnica utilizada para evaluar los resultados de un análisis estadístico y garantizar que son independientes de la partición entre datos de entrenamiento y prueba. Consiste en repetir y calcular la media aritmética obtenida de las medidas de evaluación sobre diferentes particiones. Se utiliza en entornos donde el objetivo principal es la predicción y se quiere estimar la precisión de un modelo que se llevará a cabo a la práctica.

Validación cruzada dejando uno fuera(Leave-one-out cross-validation LOOCV)

implica separar los datos de forma que para cada iteración tengamos un solo dato de prueba y todo el resto de los datos para entrenamiento.La estimación del error NO tiende a ser muy variable dependiendo de cuáles datos quedan en la tabla de aprendizaje y cuáles en la tabla de testing, es decir, la estimación del error es mucho más estable. NO se tiende a sobrestimar la estimación del error, es decir, es mucho mayor el error en la tabla de testing que en toda la tabla de datos.

Validación cruzada usando K grupos(K-fold cross-validation)

los datos se dividen en K subconjuntos (folds). Uno de los subconjuntos se utiliza como datos de prueba y el resto (K-1) como datos de entrenamiento. El proceso de validación cruzada es repetido durante K iteraciones, con cada uno de los posibles subconjuntos de datos de prueba, El error se calcula como la media aritmética de los errores de cada iteración para obtener un único resultado

Validación cruzada aleatoria

En este segundo enfoque, el bloque de validación se escoge aleatoriamente, repitiéndose el proceso k veces (siendo tanto el tamaño de cada bloque como el número k cifras arbitrarias). La ventaja de este método es que el número de iteraciones (k) no depende del tamaño de los subbloques que se consideren. La desventaja es que no es posible asegurar que todas las muestras van a ser consideradas como parte de los conjuntos de entrenamiento o de validación (pues habrá muchas que no sean escogidas nunca), y que habrá muestras que puedan ser consideradas en más de una iteración.

Margen duro Si los datos de entrenamiento son linealmente separables, podemos seleccionar dos hiperplanos paralelos que separen las dos clases de datos, de modo que la distancia entre ellos sea lo más grande posible.

Margen suave Como la mayoría de los datos del mundo real no son completamente separables linealmente, permitiremos que ocurra alguna violación de margen, lo que se denomina clasificación de margen flexible. La violación del margen significa elegir un hiperplano, que puede permitir que algunos puntos de datos permanezcan en el lado incorrecto del hiperplano y entre el margen y el lado correcto del hiperplano.

Para encontrar el margen máximo , necesitamos maximizar el margen entre los puntos de datos y el hiperplano.

Maximizar el margen Escribimos la ecuacion lineal como $y = ax+b$ como

$\mathbf{w}^T\mathbf{x}+b = 0$ donde $\mathbf{w}$ es el vector de pesos y $b$ es el sesgo, w

$(\mathbf{x}^++\mathbf{x}^-)\cdot\mathbf{x}.\hat{\mathbf{w}}^ = (\mathbf{x}^++\mathbf{x}^-)\frac{\mathbf{w}}{||w||} = \mathbf{x}^+\cdot\frac{\mathbf{w}}{||w||}-\mathbf{x}^-\cdot\frac{\mathbf{w}}{||w||}$

Maximizar-Margen es equivalente a Minimizar Pérdidas. n el algoritmo SVM, buscamos maximizar el margen entre los puntos de datos y el hiperplano. La función de pérdida que ayuda a maximizar el margen es la pérdida de bisagra(Hinge loss).

$L(w) =\sum_{i=1} max(0,1 - y_1 [w^Tx_i+b])+ \lambda= {||w||}^{2}_{2}$

$\lambda= 1/C$,$C$ siempre se usa para el coeficiente de regularización. La función del primer término, pérdida de bisagra, es penalizar las clasificaciones erróneas. Mide el error debido a una clasificación errónea (o puntos de datos que están más cerca del límite de clasificación que del margen). El segundo término es el término de regularización, que es una técnica para evitar el sobreajuste al penalizar los coeficientes grandes en el vector solución. El λ (lambda) es el coeficiente de regularización, y su función principal es determinar el compromiso entre aumentar el tamaño del margen y asegurarse de que xi se encuentra en el lado correcto del margen.

Podemos derivar la fórmula para el margen a partir de la pérdida de bisagra . Si un punto de datos está en el margen del clasificador, la pérdida de bisagra es exactamente cero. Por lo tanto, en el margen, tenemos:

$max(0,1 - y_1 [w^Tx_i+b]) = 0 \Rightarrow y_1 [w^Tx_i+b]= 1 $

Hay que tener en cuenta $y_i$ es +1 o -1, tenemos:

$\mathbf{x}^+\cdot\frac{\mathbf{w}}{||w||}-\mathbf{x}^-\cdot\frac{\mathbf{w}}{||w||} = \frac{\mathbf{1-b}}{||w||}- \frac{\mathbf{-b-1}}{||w||} = \frac{2}{||w||}$

Para minimizar dicha función de objeción, deberíamos utilizar el multiplicador de Lagrange.

Q48. Explique el algoritmo del árbol de decisión en detalle. (Explain Decision Tree algorithm in detail.)¶

El árbol de decisiones es una técnica de aprendizaje supervisado que se puede utilizar tanto para problemas de clasificación como de regresión, pero sobre todo se prefiere para resolver problemas de clasificación. Es un clasificador estructurado en árbol, donde los nodos internos representan las características de un conjunto de datos, las ramas representan las reglas de decisión y cada nodo hoja representa el resultado.

En un árbol de decisión, hay dos nodos, que son el nodo de decisión y el nodo hoja. Los nodos de decisión se utilizan para tomar cualquier decisión y tienen múltiples ramas, mientras que los nodos hoja son el resultado de esas decisiones y no contienen más ramas.

Las decisiones o la prueba se realizan sobre la base de las características del conjunto de datos dado.

Se le llama árbol de decisión porque, similar a un árbol, comienza con el nodo raíz, que se expande en más ramas y construye una estructura similar a un árbol.

Para construir un árbol, usamos el algoritmo CART, que significa algoritmo de árbol de clasificación y regresión.

Un árbol de decisiones simplemente hace una pregunta y, según la respuesta (Sí / No), divide el árbol en subárboles.

El siguiente diagrama explica la estructura general de un árbol de decisiones:

Nodo raíz: el nodo raíz es desde donde comienza el árbol de decisiones. Representa el conjunto de datos completo, que además se divide en dos o más conjuntos homogéneos.

Nodo de hoja: los nodos de hoja son el nodo de salida final y el árbol no se puede segregar más después de obtener un nodo de hoja.

Dividir: Dividir es el proceso de dividir el nodo de decisión / nodo raíz en subnodos de acuerdo con las condiciones dadas.

Rama / Subárbol: Un árbol formado al dividir el árbol.

Poda: la poda es el proceso de eliminar las ramas no deseadas del árbol.

Nodo principal / secundario: el nodo raíz del árbol se denomina nodo principal y los otros nodos se denominan nodos secundarios.

En un árbol de decisión, para predecir la clase del conjunto de datos dado, el algoritmo comienza desde el nodo raíz del árbol. Este algoritmo compara los valores del atributo raíz con el atributo de registro (conjunto de datos real) y, según la comparación, sigue la rama y salta al siguiente nodo.

Para el siguiente nodo, el algoritmo vuelve a comparar el valor del atributo con los otros subnodos y avanza. Continúa el proceso hasta que llega al nodo de la hoja del árbol. El proceso completo se puede comprender mejor utilizando el siguiente algoritmo:

Paso 1: Comience el árbol con el nodo raíz, dice S, que contiene el conjunto de datos completo.

Paso 2: Encuentre el mejor atributo en el conjunto de datos usando la Medida de selección de atributos (ASM).

Paso 3: Divida la S en subconjuntos que contengan valores posibles para los mejores atributos.

Paso 4: Genere el nodo del árbol de decisión, que contiene el mejor atributo.

Paso 5: Cree de forma recursiva nuevos árboles de decisión utilizando los subconjuntos del conjunto de datos creado en el paso -3. Continúe este proceso hasta que se alcance una etapa en la que no pueda clasificar los nodos más y se llame al nodo final como un nodo hoja.

Q57. ¿Cómo se pueden tratar los valores atípicos? (How can outlier values be treated?)¶

Los algoritmos de aprendizaje automático son muy sensibles al rango y distribución de los valores de los atributos. Los datos atípicos pueden estropear y engañar el proceso de entrenamiento, lo que da como resultado tiempos de entrenamiento más largos, modelos menos precisos y, en última instancia, resultados más pobres.

Método univariante

Uno de los métodos más simples para detectar valores atípicos es el uso de diagramas de caja. Un diagrama de caja es una presentación gráfica para describir la distribución de los datos. Los diagramas de caja utilizan la mediana y los cuartiles inferior y superior.

El método de Tukey define un valor atípico como aquellos valores del conjunto de datos que caen lejos del punto central, la mediana. La distancia máxima al centro de los datos que se permitirá se denomina parámetro de limpieza. Si el parámetro de limpieza es muy grande, la prueba se vuelve menos sensible a los valores atípicos. Por el contrario, si es demasiado pequeño, muchos valores se detectarán como valores atípicos.

Método multivariado

El método multivariado intenta resolver eso construyendo un modelo usando todos los datos disponibles y luego limpiando aquellas instancias con errores por encima de un valor dado.

Error de Minkowski

A diferencia de los métodos univariante y multivariante, no detecta ni limpia los valores atípicos. En cambio, reduce el impacto que tendrán los valores atípicos en el modelo.

El error de Minkowski es un índice de pérdida que es más insensible a los valores atípicos que el error estándar de suma cuadrada. El error de suma al cuadrado eleva cada error de instancia al cuadrado, lo que hace una contribución demasiado grande de valores atípicos al error total. El error de Minkowski lo resuelve elevando cada error de instancia a un número menor que 2, por ejemplo 1,5. Esto reduce la contribución de los valores atípicos al error total. Por ejemplo, si un valor atípico tiene un error de 10, el error al cuadrado para esa instancia será 100, mientras que el error de Minkowski será 31,62.

Método del codo La idea básica de los algoritmos de clustering es la minimización de la varianza intra-cluster y la maximización de la varianza inter-cluster. Es decir, queremos que cada observación se encuentre muy cerca a las de su mismo grupo y los grupos lo más lejos posible entre ellos.

El método del codo utiliza la distancia media de las observaciones a su centroide. Es decir, se fija en las distancias intra-cluster. Cuanto más grande es el número de clusters k, la varianza intra-cluster tiende a disminuir. Cuanto menor es la distancia intra-cluster mejor, ya que significa que los clústers son más compactos. El método del codo busca el valor k que satisfaga que un incremento de k, no mejore sustancialmente la distancia media intra-cluster

Análisis de la silueta: El análisis de la silueta mide la calidad del agrupamiento o clustering. Mide la distancia de separación entre los clústers. Nos indica como de cerca está cada punto de un clúster a puntos de los clústers vecinos. Esta medida de distancia se encuentra en el rango $[-1, 1]$. Un valor alto indica un buen clustering.

Los coeficientes de silueta cercanos a +1 indican que la observación se encuentra lejos de los clústers vecinos. Un valor del coeficiente de 0 indica que la observación está muy cerca o en la frontera de decisión entre dos clústers. Valores negativos indican que esas muestras quizás estén asignadas al clúster erróneo.

El método de la silueta calcula la media de los coeficientes de silueta de todas las observaciones para diferentes valores de k. El número óptimo de clústers k es aquel que maximiza la media de los coeficientes de silueta para un rango de valores de k.

El coeficiente de la silueta es calculado como:

$s = \frac{b-a}{max(a,b)}$

siendo a la distancia media intra-clúster y b la distancia media a las observaciones del clúster más cercano.

Q80. ¿Cuáles son las diferentes capas en CNN? (What Are the Different Layers on CNN?)¶

Convolutional Layer

Esta capa es la primera capa que se utiliza para extraer las diversas características de las imágenes de entrada. En esta capa, la operación matemática de convolución se realiza entre la imagen de entrada y un filtro de un tamaño particular MxM. Al deslizar el filtro sobre la imagen de entrada, el producto escalar se toma entre el filtro y las partes de la imagen de entrada con respecto al tamaño del filtro (MxM).

La salida se denomina mapa de características que nos brinda información sobre la imagen, como las esquinas y los bordes. Posteriormente, este mapa de características se envía a otras capas para conocer otras características de la imagen de entrada.

Pooling Layer

En la mayoría de los casos, una capa convolucional va seguida de una capa de agrupación. El objetivo principal de esta capa es disminuir el tamaño del mapa de características convolucionado para reducir los costos computacionales. Esto se realiza disminuyendo las conexiones entre capas y opera de forma independiente en cada mapa de características. Según el método utilizado, existen varios tipos de operaciones de agrupación.

La suma total de los elementos de la sección predefinida se calcula en Sum Pooling. La capa de agrupación suele servir como puente entre la capa convolucional y la capa FC.

Fully Connected Layer

La capa Fully Connected (FC) consta de los pesos y sesgos junto con las neuronas y se utiliza para conectar las neuronas entre dos capas diferentes. Estas capas generalmente se colocan antes de la capa de salida y forman las últimas capas de una arquitectura CNN.

En esto, la imagen de entrada de las capas anteriores se aplana y se alimenta a la capa FC. El vector aplanado luego se somete a unas pocas capas FC más donde suelen tener lugar las operaciones de las funciones matemáticas. En esta etapa comienza a tener lugar el proceso de clasificación.

El descenso de gradientes es un algoritmo de optimización que se usa comúnmente para entrenar modelos de aprendizaje automático y redes neuronales . Hasta que la función sea cercana o igual a cero, el modelo continuará ajustando sus parámetros para producir el menor error posible. Una vez que los modelos de aprendizaje automático están optimizados para su precisión, pueden convertirse en herramientas poderosas para aplicaciones de inteligencia artificial (IA) y ciencias de la computación.

Q92. ¿Cuál es el papel de la función de activación? (Activation Function)¶

Las funciones de activación conocidas que se utilizan en la ciencia de datos incluyen la función de unidad lineal rectificada ( ReLU ) y la familia de funciones sigmoideas , como la función sigmoidea logística, la tangente hiperbólica y la función arcangente.

Si bien este programa es bastante lento en redes con amplias capas de detección de características, es rápido en redes con una sola capa de detectores de características, llamadas " máquinas de Boltzmann restringidas ". as máquinas de Boltzmann resuelven dos problemas de aprendizaje profundo separados pero cruciales :

Q97. ¿Cuál es la diferencia entre el descenso de gradiente por lotes y el descenso de gradiente estocástico? (Batch Gradient Descent and Stochastic Gradient Descent)¶

Batch Gradient Descent

En Batch Gradient Descent, todos los datos de entrenamiento se tienen en cuenta para dar un solo paso. Tomamos el promedio de los gradientes de todos los ejemplos de entrenamiento y luego usamos ese gradiente medio para actualizar nuestros parámetros. Así que eso es solo un paso de descenso de gradiente en una época. El descenso de gradiente por lotes es ideal para múltiples errores convexos o relativamente suaves. En este caso, nos movemos algo directamente hacia una solución óptima.

Stochastic Gradient Descent

En Batch Gradient Descent estábamos considerando todos los ejemplos para cada paso de Gradient Descent. Pero, ¿qué pasa si nuestro conjunto de datos es muy grande? Los modelos de aprendizaje profundo anhelan datos. Cuantos más datos, más posibilidades hay de que un modelo sea bueno. Supongamos que nuestro conjunto de datos tiene 5 millones de ejemplos, entonces, solo para dar un paso, el modelo tendrá que calcular los gradientes de los 5 millones de ejemplos. Esta no parece una forma eficaz. Para abordar este problema tenemos el Descenso de gradiente estocástico. En Stochastic Gradient Descent (SGD), consideramos solo un ejemplo a la vez para dar un solo paso. Realizamos los siguientes pasos en una época para SGD:

1-Toma un ejemplo

2-Aliméntelo a la red neuronal

3-Calcular su gradiente

4-Utilice el gradiente que calculamos en el paso 3 para actualizar los pesos.

5-Repite los pasos 1 a 4 para todos los ejemplos del conjunto de datos de entrenamiento.

Dado que estamos considerando solo un ejemplo a la vez, el costo fluctuará con los ejemplos de capacitación y no necesariamente disminuirá. Pero a la larga, verá que el costo disminuye con las fluctuaciones.
